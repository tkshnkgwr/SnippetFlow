# SnippetFlow (SnippetManager) パフォーマンス・フットプリント計測記録

本ドキュメントでは、アプリケーションのフットプリント（実行バイナリサイズ、CPU/メモリリソース使用量）の計測結果を記録します。

---

## 1. ビルドプロファイルとバイナリサイズ

リリースビルドで生成されたバイナリサイズおよび最適化フラグの構成を記録します。
* **ビルドコマンド (egui版)**: `cargo build --release`
* **生成バイナリ (egui版)**: `target/release/snippet_manager.exe`
* **バイナリサイズ (egui版)**: **4.15 MB** (4,355,072 bytes)
* **ビルドコマンド (Tauri版)**: `npx tauri build --no-bundle`
* **生成バイナリ (Tauri版)**: `src-tauri/target/release/app.exe`
* **バイナリサイズ (Tauri版)**: **8.28 MB** (8,684,032 bytes)

## 2. メモリフットプリント (RAM)

Windowsのタスクマネージャー、またはリソースモニターによる常時稼働時のメモリ使用量です。
※ `settings.json` の新規導入に伴い、設定情報のメモリ保持が追加されましたが、実RAMへの影響は無視できる範囲（1KB未満）です。

| 動作状態 | メモリ使用量 (実物理メモリ/Working Set) | 備考 |
| :--- | :--- | :--- |
| **起動直後 (アイドル状態)** | **約 12.7 MB** (Private: 約 8.4 MB) | 日本語フォント (TTC) の動的メモリロード、および JSON ファイルI/O処理を伴います。 |
| **検索・閲覧中 (アクティブ状態)** | **約 12.7 〜 13.2 MB** | インクリメンタルフィルタリングの計算負荷がかかります。 |
| **スニペット新規追加 (フォーム入力中)** | **約 12.7 〜 13.2 MB** | キーワードの出現頻度分析およびタグ推薦計算が走ります。 |

## 3. CPU使用率

* **アイドル状態**: **0%**
  - アプリケーションが非アクティブ、またはユーザー操作が行われていないとき、フレーム描画更新は1秒に1回に抑えられるため、CPUのスパイクは発生しません。
* **アクティブ状態 (ユーザー入力・連続コピー等)**: **0.1% 〜 1.0%**
  - キーボード入力やクリックなどのイベント駆動による再描画時のみCPUが一時的に動作します。

---

## 4. ビルド最適化オプション (Cargo.toml 提示)

バイナリサイズをさらに削減し、実行パフォーマンスを最適化するための設定プロファイル（`Cargo.toml`）です。

```toml
[profile.release]
opt-level = "z"      # サイズ最優先の最適化
lto = true           # Link-Time Optimization の有効化
codegen-units = 1    # コードジェネレータのユニット数を1にして最適化を保護
panic = "abort"      # パニック時の巻き戻しコードを除外してサイズ削減
strip = true         # デバッグシンボルとシンボルテーブルの自動削除
```
