# 定型文クリップボード・マネージャー 技術・機能仕様書 (SPEC.md)

本ドキュメントは、「定型文クリップボード・マネージャー」（Rust / egui & React）の仕様、画面設計、技術スタック、および最適化仕様について定義します。

---

## 1. アプリケーション概要
* **目的**: ビジネスメールや定型業務で多用される定型テキスト（挨拶、日程、謝罪文など）をローカルに安全に蓄積し、瞬時に呼び出してクリップボードにコピーできる超軽量デスクトップ・ユーティリティ。
* **ターゲット環境**: Windows 10 / 11 (低リソース環境 / メモリ・CPU制限下)
* **主要特性**:
  * 常時最前面 (Always on Top)
  * ウィンドウタイトルバー非表示 (Decorated: false)
  * ウィンドウ背景透過 (Transparent: true)
  * システムトレイ・タスクバーの挙動に配慮した設計

---

## 2. 技術スタック

### React (Web開発 / プロトタイプ検証用)
* **Core**: React 19 (TypeScript), Vite 6
* **Styling**: Vanilla CSS, TailwindCSS v4 (ダークモード対応)
* **Icons**: lucide-react

### Rust (デスクトップ製品版)
* **GUI Engine**: `egui` / `eframe` (v0.22.0)
* **Clipboard I/O**: `arboard` (v3.2)
* **Serialization**: `serde` (v1.0), `serde_json` (v1.0)
* **DateTime**: `chrono` (v0.4)
* **Native Dialogs**: `rfd` (v0.12) - OS標準のファイルダイアログ呼び出し用
* **Cargo Features**: `windows_desktop` (Win32 API/Single Instance 等の Windows 固有機能連携用。`common_lib` の機能フラグと連動)

---

## 3. データモデル (Data Schema)

### 3.1. スニペットデータ (`snippets.json`)
定型文のデータ構造は React / Rust 共通で以下の構成を取ります。
* `id` (usize): ユニークナンバー
* `title` (String): 定型文タイトル
* `content` (String): 定型文本文
* `description` (String): 定型文の補足・説明
* `created_at` (String): 作成日時 (`YYYY-MM-DD HH:MM:SS`)
* `updated_at` (String): 最終更新日時 (`YYYY-MM-DD HH:MM:SS`)
* `deleted_at` (Option<String>): 論理削除日時 (`YYYY-MM-DD HH:MM:SS`)
* `is_deleted` (bool): 論理削除フラグ
* `tags` (Vec<String>): スニペットに紐付けられたタグリスト
* `is_pinned` / `isPinned` (bool): ピン留めフラグ（お気に入り）
* `copy_count` / `copyCount` (usize): コピーされた累計回数
* `saved_time_sec` / `savedTimeSec` (usize): 累計短縮時間（秒）

※ Tauri版およびegui版ともに、データはカレントディレクトリの `snippets.json` に保存され、リアルタイムに共有・同期されます。ブラウザ環境（プロトタイプ）でのみ `localStorage` (`snippets_db`) に保存されます。

### 3.2. アプリ設定データ (`settings.json` - Rust版のみ)
アプリケーションの動作・表示設定を保持します。
* `is_dark_mode` (bool): テーマ設定（true = ダークモード / false = ライトモード）

## 4. 各画面の機能仕様

### 4.1. 一覧画面 (定型文一覧 / Home)
* **スニペット一覧**: 登録されている定型文がリストカード形式で表示されます。
* **お気に入り（ピン留め）機能**:
  * スニペットカード上の「📌」ボタンでピン留めを切り替えることができます。
  * ピン留めされたスニペットは、選択したソート条件に関わらず常にリストの最上部に固定表示され、カード全体の境界線および背景色がIndigo調で強調表示されます。
* **過去データ（削除済み）の表示切り替え**:
  * 「過去・削除済みを表示する」のトグルをONにすると、論理削除された (`is_deleted == true`) スニペットも薄い赤色の打ち消し線付き表記で一覧にロードされます。
* **並び替え（ソート）機能**:
  * スニペット一覧を「更新日（新しい順）」「更新日（古い順）」「作成日（新しい順）」「タイトル順」「よく使う順（コピー数）」で並び替え可能です。
  * 選択したソート順は、Rust版では `settings.json`、React版では `localStorage` に自動的に保存され、永続化されます。
* **二重フィルター検索とキーワードハイライト**:
  * **テキスト検索**: タイトル、本文、説明の部分一致インクリメンタル検索。
  * **タグ検索**: 指定されたタグ文字列の部分一致検索。
  * **キーワードハイライト**: 検索バーにテキストが入力されている場合、一致した部分がカラーハイライト（黄色背景）で強調表示されます。
* **タグクラウドUI絞り込み**:
  * スニペットから一意なタグを自動抽出してボタン形式で一覧化。
  * タグをクリックすることで、トグルによる即座なタグ絞り込みが可能です。
* **クイックコピー**: 各行の「📋 コピー」ボタンで即座にクリップボードに格納され、3秒間上部に完了通知が表示されます。
* **複数選択**:
  * **結合してコピー**: 複数選択された状態で「🔗 結合してコピー」をクリックすると、選択した定型文の本文が改行（区切り文字）で結合されて一括コピーされます。
  * **差分比較**: ちょうど2つの定型文を選択している際のみ有効化され、差分比較画面へ遷移します。

### 4.2. 変更画面 (新規追加・編集 / Form)
* **新規追加**: タイトル、本文、説明、タグを入力して保存。作成日時および更新日に現在時刻を自動セット。
* **編集**: 既存内容をロードして編集。更新日に現在時刻を自動セット。編集モード時は、ID、作成日、更新日のメタデータを明示表示します。
* **削除／復元／永久削除アクション**:
  * スニペットが削除されていない（`is_deleted == false`）場合: 「🗑️ 定型文を削除」ボタン（論理削除）を表示。
  * スニペットが既に論理削除されている場合: 「🔄 アーカイブから復元」ボタンおよび「🗑️ 完全に削除する」（物理削除）ボタンを表示。
* **インテリジェント・タグ提案**:
  * 入力中の「タイトル（重み2倍）」「本文」「説明」に含まれるワードと既存タグをリアルタイム分析。
  * おすすめタグを最大5つまで自動提示し、ワンクリックで付与予定タグに追加できます。

### 4.3. 差分比較画面 (Compare)
* 選択された2つの定型文を左右に分割されたカラムに並べて表示します。
* 「タイトル」「説明」「本文プレビュー」を左右並行で閲覧・比較可能。
* **動的差し替え**: 上部のドロップダウン（コンボボックス）から、比較するスニペットをその場で動的に変更できます。
* **左右入れ替え**: 「⇄ 左右入れ替え」により、比較元(A)と比較先(B)の配置を即座に入れ替えられます。
* **差分分析ビューアー (LCS)**:
  * LCS（最長共通部分列）アルゴリズムに基づき、行ごとの追加行（緑色表記）および削除・変更行（赤色表記）をカラーハイライト表示します。

### 4.4. 複数結合画面 (Merge)
* 選択した複数の定型文をマージする専用画面。
* **追加と除外**: 結合するスニペットをチェックボックスでその場で追加・除外可能。
* **順序調整**: 選択されたスニペットの結合順序を「↑」「↓」ボタンで上下に変更できます。
* **区切り文字の選択**: 改行1つ、改行2つ、区切り線（`---`）、区切り線（`===`）、読点（`、`）、区切りなしの6種類から選択可能。
* **ライブプレビューと一括コピー**: 結合結果をプレビュー表示し、ワンクリックで結合テキストをクリップボードにコピー可能。

### 4.5. 性能メーター画面 (Performance)
* **データベース診断**: 有効件数や削除済み件数、推定されるJSONファイルサイズ、直近の検索クエリ実行時間（ミリ秒）を表示。
* **使用統計（アナリティクス）**:
  * コピー総回数および、累計でどれだけの作業時間を短縮できたかを可視化（コピーされた文字数 × 0.3秒として算出）。
  * コピー数が多い定型文トップ3のランキングを表示し、それぞれが何回コピーされ、何秒短縮したかを可視化。
* **平均速度ベンチマーク**: 検索処理を100回試行した際の平均処理時間を計測・表示。
* **大量データ負荷テスト**: 1,000件、2,000件、5,000件の擬似データ（ダミー定型文）を一括生成・一括クリーンアップする機能を搭載。メモリ負荷や描画負荷の検証に使用。
* **バックアップと復元**: egui単体版およびTauriデスクトップアプリ版の双方において、OS標準 of ネイティブファイルダイアログ（`rfd`クレート）を使用し、安全にJSONデータのインポート（復元）およびエクスポート（バックアップ）を行います。Tauri版ではブラウザ側のダウンロード制限を回避するため、Rustバックエンドに専用コマンドを定義してフロントエンドから呼び出す設計となっています。

## 5. パフォーマンスとリソース最適化仕様
常時起動されるツールであるため、システムリソース（CPU/メモリ）の消費を極限まで抑えるよう設計されています。

* **UIレイアウト・テーマ仕様**:
  - **固定ヘッダー・フッター**: リストの縦スクロール時にも、ヘッダー（タイトル・ナビゲーション）とフッター（複数選択アクションバー）が画面の定位置に固定され、中央の定型文リスト領域のみがスクロールするレイアウトを適用（egui / React版共通）。
  - **カスタムテーマフレームの導入 (egui)**: `ui.group` がライトモード切り替え時に一部ダークモードの配色を引きずってしまう問題を回避するため、すべての枠線付き要素（定型文カード、検索ボックス枠、編集メタ枠、比較/結合枠など）に対し、明示的に切り替わるカスタム `theme_card_frame` を適用（ライトモード時は純白背景に薄いグレー枠、ダークモード時は Slate 800 背景に Slate 700 枠）。
  - **ウィンドウの伸縮（リサイズ）と折り返し**: egui版においてOS標準のタイトルバー枠（デコレーション）を表示しリサイズを有効化。ユーザーが自由にウィンドウサイズを変更できるよう調整。また、横幅伸縮時にタイトルや説明文が見切れないよう、ラベルの自動テキスト折り返し（`wrap(true)`）を実装。
* **低リソース描画（描画更新制限）**:
  - `egui` の即時モード（Immediate Mode）による高負荷を防ぐため、1秒（`1000ms`）に1回しか描画更新を要求しない制約を課しています。
  - マウスホバーや入力などのインタラクションイベント発生時のみ追加で描画が走り、アイドル時のCPU使用率はほぼ **0.0%〜0.1%** を維持します。
* **クローズ処理**:
  - OS標準のタイトルバー操作によるクローズに加えて、ヘッダーに配置された「× 閉じる」ボタンのクリックによって、egui版では `std::process::exit(0)`、Tauri版では Tauri v2 WebviewWindow API (`getCurrentWebviewWindow().close()`) を呼び出し、安全にプロセスまたはウィンドウを終了します。
* **日本語フォントの自動登録**:
  - 起動時に Windows システムの標準日本語フォント（メイリオ等）を自動探索し、優先フォントとして登録することで、日本語の文字化け（豆腐文字）を完全に防止します。

## 6. CI/CD および自動リリース仕様

開発効率とリリースの正確性を保証するため、GitHub Actions による自動ビルド・デプロイ環境を構築しています。

* **継続的インテグレーション (CI - `ci.yml`)**:
  - `main` ブランチへのプッシュおよびプルリクエストをトリガーとして実行。
  - フロントエンド（Node.js）のビルドチェック。
  - Rust（egui単体版およびTauri版双方）における `cargo fmt --check`、`cargo clippy`、`cargo test` による静的解析およびテストのパスを強制。
* **自動リリースアセットパッケージ (CD - `release.yml`)**:
  - `v*` 形式のタグがプッシュされた際にトリガーとして実行。
  - Windows 環境（`windows-latest`）上でビルドを実行。
  - **Tauri アプリ**: `tauri-apps/tauri-action` を介して、自動で GitHub ドラフトリリースを作成し、インストーラー（MSI / NSIS等）および実行ファイルをアップロード。
  - **egui 単体版アプリ**: `cargo build --release` で生成される `snippet_manager.exe` を Windows-x64 向けに zip アーカイブ化し、同ドラフトリリースへ自動追加アップロード。
