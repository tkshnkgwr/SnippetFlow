# セキュリティポリシー (SECURITY.md) - SnippetFlow

本文書は、`SnippetFlow` プロジェクトにおけるセキュリティ設計原則、サポート対象バージョン、および脆弱性報告手順をまとめたドキュメントです。

---

## 1. セキュリティ設計と安心性

`SnippetFlow` は以下の設計原則により、高いセキュリティと信頼性を保証しています。

1. **ローカル完結型の動作**:
   - アプリケーションは外部サーバーと通信を行わず、登録された定型文データ（`snippets.json`）や設定（`settings.json`）はすべてユーザーのローカル環境に保存されます。
   - バックアップのインポートやエクスポートも、OSネイティブのファイルダイアログを介して完全にローカルで完結します。
2. **メモリ安全性**:
   - バックエンドおよびegui単体版は Rust 言語で構築されており、強力な所有権システムとコンパイル時のメモリ管理により、バッファオーバーフローやヌルポインタ参照などの深刻なメモリ脆弱性を排除しています。
3. **サンドボックスと安全なブリッジ**:
   - Tauri環境では、フロントエンド（React/HTML）とバックエンド（Rust）が安全な IPC（プロセス間通信）ブリッジを介してのみ通信します。Webviewコンテキストは隔離されており、不正なシステムコマンドの実行を防ぎます。
4. **XSS（クロスサイトスクリプティング）の防止**:
   - フロントエンド（React）におけるすべてのテキスト表示や差分比較画面（LCS）では、Reactの標準エスケープ処理および安全なマークアップレンダリングを徹底し、入力値に悪意あるスクリプトが混入した場合の実行を防止しています。

## 2. サポート対象バージョン

以下のバージョンに対してセキュリティアップデートが提供されます：

| バージョン              | サポート状況 |
| :---------------------- | :----------: |
| 最新リリース (`v1.9.x`) | ✅ サポート  |
| 古いリリース            | ❌ 非サポート |

---

## 3. 脆弱性の報告方法

`SnippetFlow` に潜在的なセキュリティ脆弱性を発見された場合は、公開 Issue を作成するのではなく、以下の手順でご報告ください。

1. **報告窓口**:
   - リポジトリの管理者（IssueやPR等を通じてコンタクト可能）に直接ご連絡いただくか、プライベートな報告方法をご利用ください。
2. **記載いただきたい内容**:
   - 影響を受ける `SnippetFlow` のバージョンとOS環境（Windows 10/11等）
   - egui版、Tauri版、またはReact Web版のいずれで発生するか
   - 脆弱性の詳細と再現手順（可能であれば検証用のJSONデータや操作手順）
3. **対応プロセス**:
   - 報告を受領後、速やかに内容の検証を行い、修正パッチを適用したアップデートバージョンをリリースします。
